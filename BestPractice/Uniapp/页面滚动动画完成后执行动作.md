# uni.pageScrollTo 行为规律与最佳实践

本文档基于本项目在 **支付宝小程序 / 微信小程序** 页面级滚动场景下的实测与代码实践整理，适用于 `navigationStyle: custom`、页面原生滚动（非 `scroll-view`）的列表页。

---

## API 简述

```ts
uni.pageScrollTo({
  scrollTop: number,      // 目标滚动位置（px），必填
  duration?: number,      // 动画时长（ms），默认 300
  selector?: string,      // 滚动到指定元素（部分平台）
  success?: () => void,
  fail?: () => void,
  complete?: () => void,  // 无论成功失败都会执行
})
```

页面需能纵向滚动；`scrollTop` 为页面顶部到当前视口顶部的距离。

---

## 实测行为规律

以`scrollTop = 200`、`duration = 200`为例

### 1. 动画过程由 `onPageScroll` 连续上报

调用 `pageScrollTo` 且 `duration > 0` 时，会触发多次 `onPageScroll`，`scrollTop` 从当前值**单调递增/递减**至目标附近。

- 步长**前大后小**，呈缓出（ease-out）曲线，并非匀速。
- 在触发success / complete 前的最后一帧可能是 `199.875` 这类浮点，**不等于**目标整数。

### 2. 回调触发时机 ≈ `duration`

| 回调 | 触发时机（实测） |
|------|------------------|
| `success` | 动画正常结束时，与 `complete` 几乎同刻 |
| `complete` | 动画结束或失败时都会触发 |
| `fail` | 滚动失败时（如页面不可滚动） |

示例：`duration: 200` 时，`success` / `complete` 约在 **211ms** 触发（多出的 ~10ms 为桥接与帧调度开销）。

### 3. `complete` 不等于「scrollTop 已稳定在目标值」

典型顺序：

1. 多条 `pageScroll`，末帧约 `199.875`
2. `success` / `complete` 触发
3. 可能再出现一条 `pageScroll 200`（系统收尾对齐）

因此：**在 `complete` 里不要假定 `scrollTop === scrollTop 参数`**，也不要立刻用 `boundingClientRect` 做高精度布局计算而不做等待。

## 动画完成后执行业务逻辑：最佳实践

项目封装：`scrollPromise`

```ts
// scrollPromise
export function scrollPromise(scrollTop: number = 0, duration: number = 300) {
  return new Promise<void>((resolve) => {
    uni.pageScrollTo({
      scrollTop,
      duration,
      complete: () => setTimeout(() => resolve(), 10)
    })
  })
}
```

**为什么用 `complete`？**  
`success` 只在成功时触发；`complete` 在动画结束（含失败）时都会触发。封装成 Promise 时，收尾应挂在 `complete` 上，避免漏掉边界情况。

**为什么在 `complete` 里再 `setTimeout(..., 10)` 才 `resolve`？**  
实测表明：`complete` 触发时，原生滚动未必已完全停稳——`onPageScroll` 末帧可能是 `199.875`，精确目标值（如 `200`）往往在 `complete` **之后**才再上报一次。若此时立刻 `await` 后面的逻辑（例如 `createSelectorQuery`、打开筛选层），容易读到动画末帧的偏差或和滚动抢布局。  
多等 **10ms** 是让原生层完成最后一帧对齐，成本低，比 `nextTick` 更合适：**页面滚动走小程序原生通道，不经过 Vue 的 DOM 更新周期**，`nextTick` 等不到滚动真正结束。

**为什么是 Promise？**  
方便 `await scrollPromise(target)` 与后续步骤串行，例如首页 `scrollToCatBanner` 滚到 `#cat-banner` 后再切 tab。

```ts
return scrollPromise(targetScrollTop, 0)
```
